## 栈处理括号匹配

利用**栈**进行括号匹配是一种经典且高效的方法，常用于编译器、表达式求值等场景。其核心思想是：**遇到左括号入栈，遇到右括号出栈并检查匹配**。

---

## **算法步骤**

1. **初始化一个空栈**。
2. **从左到右遍历字符串中的每个字符**：
    - 如果遇到**左括号**（如 `(`、`[`、`{`），**入栈**。
    - 如果遇到**右括号**（如 `)`、`]`、`}`）：
        1. 先判断栈是否为空。若为空，说明没有对应的左括号，**匹配失败**。
        2. 若不为空，弹出栈顶元素，检查弹出的左括号与当前右括号是否为同一类型（如 `(` 对应 `)`）。
        3. 若类型不匹配，**匹配失败**。
3. **遍历结束后**，如果栈为空，说明所有括号都已正确配对，**匹配成功**；否则，**匹配失败**。

---

## **示例代码（Python）**

````python
def is_valid_parentheses(s):
    stack = []
    pairs = {')': '(', ']': '[', '}': '{'}
    for ch in s:
        if ch in '([{':
            stack.append(ch)
        elif ch in ')]}':
            if not stack or stack.pop() != pairs[ch]:
                return False
    return not stack

# 示例
print(is_valid_parentheses("([{}])"))  # 输出 True
print(is_valid_parentheses("([)]"))    # 输出 False
````

---

## **原理说明**

- **栈的“后进先出”特性**，保证了最近遇到的左括号必须最先被匹配。
- 只要有不匹配或栈提前为空，立即判定失败。
- 最终栈为空，说明所有括号都一一配对。

---

**总结**：利用栈可以高效、简洁地实现括号匹配，适用于各种嵌套和多类型括号的场景。

## B树 (B-Tree)

B树是一种**自平衡的多路搜索树**，它不是二叉树。它被专门设计用于处理存储在磁盘等外部存储设备上的大量数据，是数据库和文件系统实现索引的常用数据结构。

### **1. B树的核心特性**

*   **多路搜索**：每个节点可以拥有多个子节点（超过2个），因此树的高度非常低，能有效减少磁盘I/O次数。
*   **自平衡**：通过节点的分裂和合并操作，B树始终保持平衡，所有叶子节点都位于同一层。
*   **节点结构**：每个节点既存储**关键字 (keys)**，也存储指向子节点的**指针 (pointers)**。关键字在节点内部是有序的。
*   **数据存储**：数据记录（或指向数据记录的指针）只存储在**叶子节点**上（这是B+树的特性，在B树中，内部节点也可以存数据，但为了简化和与B+树对比，通常这样理解）。

### **2. 4阶B树是什么？**

B树的“阶” (Order)，通常用 `m` 表示，定义了一个节点**最多可以拥有的子节点数量**。

因此，一个**4阶B树 (m=4)** 意味着：

*   **最大子节点数**：每个节点最多有 **4** 个子节点。
*   **最大关键字数**：每个节点最多有 `m-1 = 3` 个关键字。
*   **最小子节点数**：除了根节点，每个非叶子节点至少有 `ceil(m/2) = ceil(4/2) = 2` 个子节点。
*   **最小关键字数**：除了根节点，每个节点至少有 `ceil(m/2) - 1 = 1` 个关键字。

> **根节点特例**：如果根节点不是唯一的节点，它至少有2个子节点。如果树只有一个根节点，它可以有少于最小值的关键字。

### **3. 如何构建一个4阶B树**

构建B树的过程就是不断插入新关键字并维持其性质的过程。核心操作是**节点分裂**。

我们以插入序列 `[10, 20, 30, 40, 50, 60, 70]` 为例来构建一个4阶B树。

**1. 插入 10, 20, 30**
*   创建一个根节点，依次插入10, 20, 30。由于最大关键字数是3，节点未满。
*   树的结构: `[10, 20, 30]`

**2. 插入 40**
*   尝试将40插入到根节点 `[10, 20, 30]`。此时节点包含 `[10, 20, 30, 40]`，关键字数量为4，超过了最大值3，需要**分裂**。
*   **分裂操作**：
    *   选择中间的关键字 `20` **向上提升**，成为新的根节点。
    *   小于 `20` 的关键字 `10` 成为左子节点。
    *   大于 `20` 的关键字 `30, 40` 成为右子节点。
*   树的结构:
    ```
          [20]
         /    \
      [10]    [30, 40]
    ```

**3. 插入 50**
*   从根节点 `[20]` 开始查找。`50 > 20`，进入右子节点 `[30, 40]`。
*   节点 `[30, 40]` 未满，直接插入50。
*   树的结构:
    ```
          [20]
         /    \
      [10]    [30, 40, 50]
    ```

**4. 插入 60**
*   从根节点 `[20]` 查找，`60 > 20`，进入右子节点 `[30, 40, 50]`。
*   该节点包含 `[30, 40, 50, 60]`，关键字数量为4，超过最大值3，需要**分裂**。
*   **分裂操作**：
    *   选择中间的关键字 `40` **向上提升**到父节点 `[20]`。
    *   父节点 `[20]` 未满，合并后变为 `[20, 40]`。
    *   小于 `40` 的关键字 `30` 成为一个新节点。
    *   大于 `40` 的关键字 `50, 60` 成为另一个新节点。
*   树的结构:
    ```
          [20, 40]
         /   |    \
      [10] [30]  [50, 60]
    ```

**5. 插入 70**
*   从根节点 `[20, 40]` 查找。`70 > 40`，进入最右侧的子节点 `[50, 60]`。
*   节点 `[50, 60]` 未满，直接插入70。
*   最终树的结构:
    ```
          [20, 40]
         /   |    \
      [10] [30]  [50, 60, 70]
    ```

这个过程展示了B树通过节点分裂来保持平衡和满足其阶数定义的动态构建方法。
