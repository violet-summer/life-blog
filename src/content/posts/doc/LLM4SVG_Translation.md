# LLM4SVG：赋能大语言模型理解与生成复杂矢量图形

**摘要**：大语言模型（LLMs）的空前进步深刻影响了自然语言处理，但尚未完全拥抱可缩放矢量图形（SVG）生成领域。虽然 LLMs 在训练过程中从网页中编码了部分 SVG 数据知识，但最近的研究表明，LLMs 内部语义模糊且标记化的表示可能导致矢量图元预测中的幻觉。此外，LLM 训练通常缺乏对矢量路径渲染顺序的建模和理解，这可能导致输出矢量图元之间的遮挡。在本文中，我们提出了 LLM4SVG，这是通过使 LLMs 能够更好地理解和生成矢量图形来弥补这一差距的初步且实质性的一步。LLM4SVG 通过可学习的语义标记促进对 SVG 组件的更深层次理解，这些标记精确地编码了这些标记及其相应的属性，以生成语义对齐的 SVG 输出。利用一系列可学习的语义标记，开发了一个用于指令遵循的结构化数据集，以支持跨两个主要任务的理解和生成。我们的方法为现有的语言大模型引入了模块化架构，集成了语义标签、矢量指令编码器、微调命令和强大的 LLMs，将几何、外观和语言信息紧密结合。为了克服 SVG-文本指令数据的稀缺性，我们开发了一个自动数据生成流水线，收集了超过 25 万个 SVG 数据和 58 万条 SVG-文本指令的大规模数据集，这促进了 LLM 开发中流行的两阶段训练策略的采用。通过探索各种训练策略，我们开发了 LLM4SVG，它显著超越了基于优化的渲染方法和基于语言模型的基线，在人类评估任务中取得了显著成果。

## 1. 引言

可缩放矢量图形（SVGs）代表了一种基本的图像编码方法，其中视觉元素由矢量路径、曲线和多边形等基本形状形成，并由数学方程定义。这与位图图形形成对比，位图图形将图像描绘为网格上的像素数组。矢量图形的关键优势之一是它们能够在不同分辨率下保持高精度和一致的视觉质量，因为它们可以无限缩放而不会丢失任何细节。此外，矢量图形通常保持紧凑的文件大小，这增强了它们的存储和传输效率。更关键的是，它们提供了卓越的可编辑性，使设计人员能够轻松选择、修改和组合元素。这一属性在设计过程中尤为重要，有助于无缝调整并促进创意探索。由于其固有属性，SVGs 特别适用于视觉流畅的设计应用。

近年来，人们对矢量图形生成的兴趣显著增加。尽管 SVGs 具有显著优势，但当前基于深度学习的生成方法在产生高质量、复杂的 SVG 输出方面仍面临局限性。目前的方法使用受限的命令路径表示 SVG 并利用序列模型学习。此类方法主要处理简化的 SVGs，局限于基本路径命令（如移动到、线到、三次贝塞尔曲线），且复杂性通常有限；某些方法专门关注基本字体或图标。最近的创新结合了先进的图像扩散模型来促进位图图像的生成，随后通过基于贝塞尔曲线表示的可微分渲染器将其转换为 SVG 格式。虽然利用生成位图图像引入了一定程度的多样性，但这一过程具有繁琐的迭代程序特征，且生成的 SVGs 仍然不可编辑，无法满足专业设计人员的期望。鉴于这些发展，在能够直接合成复杂且详细的 SVG 代码、充分利用复杂设计应用所需的全套 SVG 图元的系统领域，仍然存在关键差距。

大语言模型（LLMs）最近的进展证明了它们理解和解析 XML 语法标的能力，这是通过对来自 Web 的各种文本语料库进行广泛预训练实现的。这种熟练程度为 LLMs 合成矢量图形奠定了坚实基础。值得注意的是，最先进的模型（如 ChatGPT 和 GPT-4）在生成简单的矢量图元（如三角形和矩形）方面表现出熟练度；然而，它们在合成复杂图形时经常遇到显著局限。这是因为在预训练期间，来自互联网的 SVG 数据通常嵌入在网页代码中，要求 LLMs 解析分层语言，这使得 SVG 数据在冗长的 XML 标签中变得难以获取。这些局限性表现为矢量路径序列中的混淆或幻觉，产生语义模糊的图形和编码不当的属性。

在本文中，我们提出了 LLM4SVG，这是通过使 LLMs 能够更好地理解和生成矢量图形来弥补这一差距的初步且实质性的一步。LLM4SVG 构建在现有的 LLM（如 GPT-2 和 LLaVA）之上，最大限度地发挥了模型在矢量图形合成方面的潜力。本文的贡献如下：

*   **引入 LLM4SVG：推进 LLM 对 SVG 的理解和生成能力**。我们提出了 LLM4SVG，这是一个旨在弥补大语言模型（LLMs）中 SVG 生成差距的先驱模型，解决了准确预测矢量图元和管理遮挡方面的挑战。通过利用可学习的语义标记，LLM4SVG 使 LLMs 能够有效地编码 SVG 组件和属性，使生成输出符合人类设计标准并降低语义模糊的风险。
*   **集成几何、视觉和语言特征的模块化架构**。LLM4SVG 采用模块化设计，通过矢量指令编码器、语义标记嵌入和用于精确坐标及颜色预测的回归头增强了传统的 LLM 架构。这种多模态集成促进了对 SVG 元素的全面理解，使 LLMs 能够通过有效融合几何、外观和语言信息来生成精炼、语义一致的矢量图形。
*   **开发大规模 SVG-文本数据集和两阶段训练策略**。针对 SVG-文本指令数据的缺乏，LLM4SVG 引入了一个自动流水线，收集了 58 万个 SVG-文本指令对。我们收集了由人类设计人员设计的 25 万个 SVG 样本。这个大规模数据集支持两阶段训练策略，增强了 LLM 在 SVG 生成任务中的理解和性能，为 LLMs 中 SVG 相关能力的进一步发展奠定了坚实基础。

## 2. 相关工作

### 2.1 矢量图形合成
生成 SVG 内容的一种方法是训练神经网络生成预定义的 SVG 命令和属性。设计用于学习 SVG 表示的神经网络通常包括 RNN、VAE 和 Transformer 等架构。这些网络的训练严重依赖于矢量形式的数据集。然而，大规模矢量数据集的有限可用性显著限制了它们的泛化能力和合成复杂矢量图形的能力。

Li 等人引入了一种可微分渲染器，桥接了矢量图形和位图图像领域。虽然传统的矢量图形生成方法需要基于矢量的数据集，但这种 SVG 生成方法涉及在预训练视觉语言模型的指导下直接优化 SVG 路径的几何和颜色参数。视觉文本嵌入对比语言-图像预训练模型（CLIP）的最新进展实现了一些成功的素描合成方法。与 CLIP 相比，几种方法将扩散模型与可微分渲染器集成，以实现卓越的生成能力和增强的图像一致性。此外，最近的研究将基于优化的方法与神经网络结合来学习矢量表示，将几何约束纳入矢量图形中。

### 2.2 大语言模型
大语言模型（LLMs）在自然语言理解方面取得了实质性进展，通过对大规模文本数据集（如 ChatGPT/GPT-4、LLaMA、Claude 3.5、Qwen 2.5、Gemini、Grok-2、Yi、Falcon、Phi 系列等）进行广泛预训练，展现出强大的泛化和推理能力，以完成现实世界的应用。

广义上，LLMs 可分为两类。第一类包括采用大语言模型与单个特定模态模型对接的模型。这种方法规避了模型训练的需求，但严重依赖于预先存在的模型或 API 的可用性和能力。第二类包括使用端到端训练策略的模型，包含两个主要范式。第一种涉及从头开始训练模型，类似于纯文本 LLMs，使用大规模多模态语料库和数据集。与其从头开始训练 LLM，不如从预训练模型开始更高效且实用。在我们的工作中，我们采用了基于预训练 LLMs 的微调策略，专注于使模型适应生成和理解矢量图形。虽然可以在需要时整合多模态能力，但我们的主要重点是利用预训练 LLM 强大的基础知识和泛化能力，这为矢量图形领域的专业化、任务导向型应用提供了坚实基础。

LLMs 的适应性也自然延伸到多模态领域。这使得它们不仅可以用于基于文本的任务，还可以作为扩展到多模态应用的基础架构。在此基础上，多模态大语言模型（MLLMs）旨在解释和生成跨各种模态的内容，包括但不限于图像、音频、动作和 3D 点云。

### 2.3 指令微调
在自然语言处理（NLP）领域，研究人员研究了各种指令微调 LLMs 的方法，旨在增强它们遵循自然语言指令和执行现实世界任务的能力。研究发现，这种简单直接的方法可以显著增强 LLMs 的零样本和少样本泛化能力。随着多模态大语言模型（MLLMs）的日益普及，LLaVA 利用视觉指令数据进行视觉指令微调，显著增强了开源 MLLMs 在多模态任务上的性能。

最近，几项研究探索了通过将 SVG 代码视为基于文本的表示，直接使用图像嵌入微调 LLMs 以生成可缩放矢量图形（SVG）。虽然这些方法在将 LLMs 应用于矢量图形生成方面显示出前景，但它们往往忽略了 SVG 文件的分层和结构化性质，将其视为顺序文本。此外，大多数 LLMs 通常将数字视为单个字符，这使得它们难以理解连续的数值数据（如坐标）。这种简化可能会限制模型充分捕获 SVG 复杂依赖关系和文件结构的能力，从而影响生成图形的质量和语义准确性。

## 3. 矢量指令遵循数据

我们开发了五种不同的指令模板，专门用于矢量图形生成和理解任务。具体而言，类型 #1 和 #2 促进生成任务，而类型 #3、#4 和 #5 侧重于理解任务。

开发端到端 LLM 的一个重大障碍是获取大规模指令遵循数据，这对于表示学习、对齐潜在空间和引导模型对齐人类意图是必不可少的。在矢量图形领域，这一障碍尤为严重，因为与矢量图形相关的高生产成本和标记挑战将当前研究限制在有限的应用范围内。这些领域包括简单的人手绘图、字体和图标图形。

为了应对这些挑战，我们手动收集了约 25 万个彩色且复杂的矢量图形，并开发了一个规范化过程，以确保收集的数据符合一致的标准，包括统一的画布大小、相对坐标系和表示。这些高质量的矢量数据集为 LLM4SVG 的开发提供了坚实基础。此外，受最近 GPT 模型在文本注释任务中成功的启发，我们利用 BLIP 对光栅化矢量图进行注释，并利用 GPT-4 进行指令遵循数据收集。

**SVG 重新标注（Re-captioning）**：从互联网收集的 SVG 数据通常包含噪声，直接用于学习可能会损害模型的潜在表示准确性。SVG 文件中约有一半的数据对于视觉渲染是冗余的。这种冗余包括：(1) 矢量编辑应用程序使用的临时数据，(2) SVG 的非最优结构表示，以及 (3) 未使用且不可见的图形元素。我们提出了一个 SVG 数据预处理流水线，旨在无损地减小矢量编辑应用程序生成的 SVG 文件的大小。该流水线结合了针对 SVG 文件四个关键方面的精心优化策略：元素、属性、路径和输出。优化 SVG 后，我们将其光栅化为 512×512 像素的图像，并使用 BLIP 模型生成相应的标题作为文本提示。因此，我们获得了一个多模态数据集，每个条目都是由优化后的 SVG、其对应的光栅化图像和 BLIP 模型生成的文本描述组成的三元组。

随后，我们提出了一种自动生成 SVG 指令遵循数据的策略。指令数据分为两个截然不同的部分：第一部分（项目 #1 和 #2）涉及矢量图形的合成，而第二部分（项目 #3、#4、#5）涉及矢量图形的理解。

总共，我们收集了一个由 25 万个带注释、高质量且标准化的矢量图形组成的数据集，以及 58 万个独特的 SVG-文本-图像样本。

## 4. 指令微调方案

### 4.1 SVG 语义标记
对于输入的 SVG $X_v$，我们将其从原始代码转换为结构化表示。为此，我们定义了 55 个 SVG 语义标记 $s_i \in \mathbb{R}^{55}$（包括 15 个标签标记、30 个属性标记、10 个路径命令标记）。这些 SVG 标记用于替换 SVG 源代码中的所有标签和属性，从而防止将 SVG 标签和属性作为常规文本进行文本编码。例如，标签 `<path>` 将被标记化为一个 SVG 语义标记，而不是分词器中的字面量“path”。这确保了 SVG 标签和属性的唯一性，并允许它们以符合 SVG 定义的方式高效集成到 LLMs 中，并优化标记初始化。我们调整嵌入层 $W_{emb} \in \mathbb{R}^{|V|'}$ 来学习这些新标记的嵌入，其中 $|V|' := |V| + 55$ 代表原始词汇量与额外 SVG 标记之和。每个新标记都根据其描述性文本的语义平均值进行初始化。

### 4.2 架构
主要目标是有效利用预训练 LLMs 和视觉模型的能力。我们选择 GPT-2、Phi-2 和 Falcon 作为我们框架的基础 LLMs。选择这些模型是因为它们具备理解视觉和文本数据的能力，并且在现有开源模型中的各种语言任务中表现出有效的指令遵循属性。理论上，具有类似能力的其他 LLMs 也可以作为我们方法的基础。

### 4.3 训练
对于每个 SVG $X_v$，我们从 SVG 指令数据集中采样多轮对话数据。我们使用 LLM 原始的自回归训练目标对 LLM 进行指令微调，以增强其在预测任务上的性能。

*   **第一阶段：特征对齐预训练**。在训练中，我们保持视觉编码器和 LLM 的权重冻结，仅专注于通过可训练参数 $\theta = W_{emb}$（词嵌入）最大化似然。
*   **第二阶段：端到端全量微调**。我们采用整个指令数据集对所有参数（包括 LLM 的参数）进行监督微调。我们考虑了两种特定的使用场景：高效参数微调（如 LoRA）和全量微调。

## 5. 实验

### 5.1 定量评估
我们的 LLM4SVG 模型在基于 LLM 的 SVG 生成方法中取得了最高性能。虽然在视觉质量方面可能无法超越基于优化的方法，但它仍然具有竞争力，在多个评估指标上与其他方法性能紧密匹配。此外，我们的方法仅需要模型推理，消除了耗时的优化过程，从而使 SVG 生成时间显著短于基于优化的方法。

由于大多数 LLMs 对连续数值数据的支持有限，这些模型生成的 SVGs 经常表现出不精确的坐标和颜色表示，通常依赖整数而非小数，并使用基本的颜色名称（如黑色或蓝色）而非更精确的十六进制代码。我们的方法通过在 SVG 路径中加入小数坐标和十六进制颜色代码解决了这些缺点，从而扩展了模型表示更广泛且更准确颜色范围的能力。

### 5.2 定性评估
我们的方法在 SVG 生成的完整性、图元的选择和放置以及矢量图形传达的语义丰富度方面优于其他基于 LLM 的方法。基于优化的方法通常采用大量重叠和交织的图元来接近真实样本，这往往导致过多的笔画冗余，且单个图元的形状在孤立观察时可能显得不规则，使其在实际应用中的实用性较低。

**人类评估**：为了评估我们生成的 SVGs 与其他 LLMs 相比的视觉效果，我们进行了一项用户研究。结果显示，我们生成的 SVGs 在提示词对齐、视觉质量方面显著优于其他基于 LLM 的方法，且我们 SVGs 的视觉质量与人类创建的 SVGs 非常接近。

### 5.3 消融实验
*   **LLM4SVG 架构分析**：LLM4SVG 的性能受基础模型的显著影响，特别是 LLM 分词器如何管理连续数字标记。例如，Qwen 模型在分词过程中将所有数字和小数点转换为单个标记，导致其无法正确处理连续数值数据。而像 GPT-2 这样将 1000 以内的所有数字枚举为分词器中单个标记的 LLMs，表现出对数值数据更强的理解力。
*   **SVG 语义标记分析**：在模型训练过程中，我们通过加入 55 个 SVG 特有的语义标记扩展了分词器。这种修改使 SVG 标签和属性能够被识别为独特的标记，从而将它们与通用文本区分开来。这有助于模型更有效、更准确地学习 SVG 生成模板。
*   **词嵌入初始化方法分析**：我们使用 T-SNE 将新添加的标记映射到二维可视化中。结果表明，使用标记的文本描述进行初始化的策略将标记嵌入聚集在特征空间中相对紧凑的区域，这有助于降低模型训练难度。

## 6. 结论与讨论

在这项工作中，我们提出了一种利用大语言模型（LLMs）进行 SVG 理解和生成的方法，使 LLMs 能够直接解释 SVG 源代码并生成高质量的 SVGs。这些生成的 SVGs 表现出一定的复杂性，并符合人类设计原则。我们的方法引入了一种结构化的 SVG 编码方法，解决了 LLMs 将 SVG 源代码视为普通文本的挑战。此外，我们收集了一个由人类设计人员设计的 25 万个高质量 SVG 和 58 万个 SVG-文本指令对组成的数据集。预计该数据集将显著加速未来与矢量图形相关的研究。

**未来工作**：我们的 LLM4SVG 可以利用不同的 LLMs 作为其基础模型。未来，更先进、更复杂的模型可以作为基础，并结合多模态大语言模型来整合图像信息。这种扩展可能有助于降低模型理解和生成的难度，实现更准确、更高效的 SVG 生成。
